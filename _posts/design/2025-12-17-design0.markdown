### 1. 전략(Strategy) 패턴

#### 요약

> 알고리즘을 캡슐화해서 실행 중에 쉽게 교체할 수 있게 하는 패턴

#### 언제 사용?

- if/else, switch가 많아질 때
- 같은 목적이지만 구현 방식이 여러 개일 때

#### 활용 사례

- 결제 방식 (카드 / 계좌이체 / 카카오페이)
- 정렬 알고리즘 선택 (QuickSort / MergeSort)
- Spring의 PasswordEncoder (BCrypt, SCrypt 등)

### 2. 옵저버(Observer) 패턴

#### 요약

> 객체 상태 변화 시, 구독자에게 자동으로 알림을 보내는 패턴

#### 언제 사용?

- 1:N 의존 관계
- 상태 변화에 따라 여러 객체가 반응해야 할 때

#### 활용 사례

- 이벤트 리스너
- Kafka, RabbitMQ Pub/Sub 구조
- 프론트엔드 상태 관리 (Redux, Vue reactivity)

### 3. 데코레이터(Decorator) 패턴

#### 요약

> 기존 객체를 수정하지 않고 기능을 동적으로 확장하는 패턴

#### 언제 사용?

- 상속이 복잡해질 때
- 기능을 조합해서 사용해야 할 때

#### 활용 사례

- Java I/O (BufferedInputStream)
- Spring AOP
- 로그, 권한 체크, 트랜잭션 처리

### 4. 팩토리 메소드(Factory Method) 패턴

#### 요약

> 객체 생성을 서브클래스에 위임하는 패턴

#### 언제 사용?

- 객체 생성 로직이 복잡하거나 변경 가능할 때

#### 활용 사례

- LoggerFactory.getLogger()
- Spring Bean 생성
- UI 컴포넌트 생성 (Button, TextField)

### 5. 추상 팩토리(Abstract Factory) 패턴

#### 요약

> 서로 관련된 객체들을 일관성 있게 생성하는 패턴

#### 언제 사용?

- 제품군이 존재할 때
- 구현체 교체가 쉬워야 할 때

#### 활용 사례

- DB 접근 계층 (MySQL / Oracle)
- UI 테마 (Windows / Mac)
- Spring의 FactoryBean

### 6. 싱글턴(Singleton) 패턴

#### 요약

> 클래스의 인스턴스를 하나만 생성하도록 보장하는 패턴

#### 언제 사용?

- 전역에서 공유되는 객체가 필요할 때

#### 활용 사례

- 설정(Config) 객체
- DB 커넥션 풀
- Spring Bean 기본 Scope

### 7. 커맨드(Command) 패턴

#### 요약

> 요청을 객체로 캡슐화하여 실행·취소·재사용 가능하게 만드는 패턴

#### 언제 사용?

- 실행 요청을 큐잉, 로그, 롤백해야 할 때

#### 활용 사례

- Undo / Redo 기능
- 작업 큐, 배치 처리
- 리모컨 버튼 구현 예제

### 8. 어댑터(Adapter) 패턴

#### 요약

> 호환되지 않는 인터페이스를 연결해주는 패턴

#### 언제 사용?

- 기존 코드를 수정할 수 없을 때
- 외부 라이브러리 연동 시

#### 활용 사례

- 레거시 API 래핑
- JDBC Driver
- DTO ↔ Entity 변환

### 9. 퍼사드(Facade) 패턴

#### 요약

> 복잡한 서브시스템을 단순한 인터페이스로 감싸는 패턴

#### 언제 사용?

- 사용자가 내부 구조를 몰라도 되게 하고 싶을 때

#### 활용 사례

- Service Layer
- API Gateway
- 라이브러리 진입 클래스

### 10. 템플릿 메소드(Template Method) 패턴

#### 요약

> 알고리즘의 뼈대는 고정하고 일부 단계만 재정의하는 패턴

#### 언제 사용?

- 전체 흐름은 동일, 일부 로직만 다를 때

#### 활용 사례

- Spring AbstractController
- JDBC Template
- 배치 처리 공통 로직

### 11. 반복자(Iterator) 패턴

#### 요약

> 내부 구조를 노출하지 않고 컬렉션을 순회하는 패턴

#### 언제 사용?

- 컬렉션 구현을 숨기고 싶을 때

#### 활용 사례

- Java Iterator
- for-each 문
- 데이터 스트림 처리

### 12. 컴포지트(Composite) 패턴

#### 요약

> 객체를 트리 구조로 구성해 단일 객체처럼 다루는 패턴

#### 언제 사용?

- 부분과 전체를 동일하게 처리해야 할 때

#### 활용 사례

- 파일 시스템 (폴더/파일)
- UI 컴포넌트 트리
- 조직도 구조

### 13. 상태(State) 패턴

#### 요약

> 상태에 따라 객체의 행동을 변경하는 패턴

#### 언제 사용?

- 상태에 따른 분기문이 많아질 때

#### 활용 사례

- 주문 상태 (결제대기 → 결제완료 → 배송중)
- 게임 캐릭터 상태
- TCP 연결 상태

### 14. 프록시(Proxy) 패턴

#### 요약

> 실제 객체 접근을 제어하는 대리 객체를 두는 패턴

#### 언제 사용?

- 접근 제어, 지연 로딩, 로깅이 필요할 때

#### 활용 사례

- Spring AOP
- Hibernate Lazy Loading
- 원격 프록시 (RPC)
